### Prompt 02: Crie a sua primeira proposta(proposal) de alteração e validá-la

```bash
Quero adicionar [SUA FUNCIONALIDADE AQUI]. Crie uma
proposta de alteração OpenSpec para esta funcionalidade
```

Ex.:

```bash
Quero adicionar registro de usuário.
Não quero recusrsos adicionais
Não quero autenticação.
Crie uma proposta de alteração OpenSpec para esta funcionalidade
```

### O que eu fiz?

Nesse ponto, simplesmente solicitei a IA para criar a funcionalidade.
O agente de ia gerará uma proposta deste no recusto solicitado.
A proposta listará os motivos para alteração.
Gerará as tarefas para execução

Arquivos Criados
proposal.md - Define o "por quê", "o quê" e o impacto da mudança
tasks.md - Checklist detalhado de tarefas
spec.md - Especificação detalhada (Requisitos e cenários (formato ADDED/MODIFIED/REMOVED))

**Dica:** Aprenda o fluxo de trabalho do OpenSpec:
Solicite no terminal doa agente:

```bash
Por favor, explique o fluxo de trabalho do OpenSpec a partir de openspec/AGENTS.md
e como devo trabalhar com você neste projeto
```

Passos:
Contexto antes de agir: ler project.md e mudanças ativas (openspec list) para evitar duplicação.
Stage 1 – Criar proposta. Validar com openspec validate <id> --strict. Não implementar antes de aprovar a proposta.

**Valide:** é importante você validar para seguir para o próximo passo.
proposal.md -
tasks.md -
spec.md -

Para validar a proposta, execute:

```bash
openspec validate [change-id] --strict
```

Ex.:
```bash
openspec validate [add-user-registration] --strict
```

atualizará: openspec/changes/
